# Example Changelog Outputs

Sample changelogs generated by openspec-generate-changelog skill.

## Example 1: Simple Release

```markdown
# Changelog

## [Unreleased]

### Added
- Add dark mode theme support (add-dark-mode)
- Implement user authentication system (add-auth)

### Changed
- Update API response handling for better error messages (update-api-errors)
- Refactor configuration loading for improved performance (refactor-config)

### Fixed
- Fix session timeout handling (fix-session-timeout)
- Resolve authentication token refresh bug (fix-token-refresh)
```

## Example 2: Breaking Change

```markdown
# Changelog

## [1.2.0] - 2024-02-15

### Added
- Implement OAuth2 authentication with multiple providers (add-oauth2)
- Add user profile management (add-profile)
- Create admin dashboard for user management (add-admin-dashboard)

### Changed
- Update user data model to support new authentication providers (update-user-model)
- Migrate existing user accounts to new auth system (migrate-users)

### Breaking
- **BREAKING**: Legacy OAuth1 endpoint `/api/v1/auth` is deprecated and will be removed in v2.0. Use `/api/v2/auth` instead.
- **BREAKING**: User data structure changed: `user.id` is now string instead of integer. Migration required for existing databases.

For migration guide, see [MIGRATION-v1-to-v2](docs/migration-v1-to-v2.md).
```

**Notes**:
- Breaking changes are clearly marked with `**BREAKING**`
- Migration guide is referenced
- Version number indicates breaking change (1.2.0 major version)

## Example 3: Infrastructure Changes

```markdown
# Changelog

## [Unreleased]

### Added
- Add CI/CD pipeline with GitHub Actions (setup-ci)
- Implement automated testing on PRs (add-tests)

### Changed
- Update project structure to follow Standard Go Layout (refactor-structure)
- Optimize build process for faster compilation (optimize-build)

### Fixed
- Fix Docker build caching issue (fix-docker-cache)
- Resolve test environment variable conflicts (fix-test-env)

### Infrastructure (non-user-facing)
- Upgrade to Go 1.21 from Go 1.20 (upgrade-go)
- Update development tooling mise to latest version (update-mise)
- Add pre-commit hooks for code quality (add-pre-commit)
```

**Notes**:
- Infrastructure changes grouped under "Infrastructure" or separate category
- Helpful for internal team context, less for end users

## Example 4: Security Fix

```markdown
# Changelog

## [1.0.1] - 2024-03-22

### Security
- **Security**: Patch JWT token validation vulnerability (patch-jwt-leak). This vulnerability could allow token expiration bypass. [CVE-2024-1234]

### Fixed
- Add input validation for email addresses (validate-email)
- Improve error messages for authentication failures (improve-auth-errors)

### Changed
- Update session management to track token refresh attempts (update-session-tracking)
```

**Notes**:
- Security fix prominently displayed first
- CVE/issue reference included
- Related fixes grouped under same release

## Example 5: Multi-Feature Release

```markdown
# Changelog

## [1.5.0] - 2024-06-30

### Added
- Add dark mode theme support with system preference detection (add-dark-mode)
- Implement user authentication system with JWT tokens (add-auth)
- Add data export functionality in CSV format (add-csv-export)
- Create user profile management interface (add-profile)
- Implement real-time notifications system (add-notifications)
- Add search functionality across users and data (add-search)

### Changed
- Refactor data access layer for better performance (refactor-data-layer)
- Update UI components for improved responsiveness (update-ui-components)
- Optimize database queries for faster loading (optimize-db)

### Fixed
- Fix session timeout on mobile devices (fix-session-timeout)
- Resolve authentication token race condition (fix-token-race)
- Fix CSV export encoding for international characters (fix-csv-encoding)
- Resolve notification delivery issues (fix-notifications)

### Removed
- Remove legacy XML export format (remove-xml-export) - deprecated in favor of CSV export

### Breaking
None

### Changed (Infrastructure)
- Update CI/CD pipeline for automated releases (setup-ci)
- Migrate to GitHub Actions from Travis CI (migrate-ci)
```

**Notes**:
- Multiple changes grouped by category
- Each category has multiple entries
- No breaking changes in this release

## Example 6: Date-Filtered Changelog

```markdown
### Added

- Add dark mode support (add-dark-mode)
- Implement user authentication (add-auth)

### Changed
Changes from September 2024:

- Update API error handling (update-api-errors)
- Refactor configuration management (refactor-config)

### Fixed
Changes from September 2024:

- Fix session timeout (fix-session-timeout)
```

**Notes**:
- Generated with `openspec-generate-changelog --since 2024-09-01 --until 2024-09-30`
- Only changes within date range included
- Each section has "Changes from [month] [year]" prefix

## Example 8: Moving Unreleased to Version

### Before Release

```markdown
# Changelog

## [Unreleased]

### Added
- Add dark mode support (add-dark-mode)
- Implement user authentication (add-auth)
- Add CSV export functionality (add-csv-export)
```

### After Release

```markdown
# Changelog

## [1.0.0] - 2024-02-12

### Added
- Add dark mode support (add-dark-mode)
- Implement user authentication (add-auth)
- Add CSV export functionality (add-csv-export)
```

**Notes**:
- Version header created: `## [1.0.0] - 2024-02-12`
- All unreleased entries moved under version
- Empty `[Unreleased]` section removed

## Example 9: Empty Archive

- Helpful for initial projects

```markdown
[Full changelog content follows project-specific format]
```
